Complete reference of all fields you can query in OQL.

## Test Case Fields

### Identifiers & Metadata

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `identifier` | string | Test case ID | `identifier ^ "TC-10"` |
| `title` | string | Test case title | `title ~ "login"` |
| `description` | string | Test description | `description ~ "authentication"` |
| `test_type` | string | manual, automated, api | `test_type = manual` |
| `status` | string | draft, active, archived | `status = active` |
| `priority` | string | p0, p1, p2, p3, p4 | `priority IN (p0, p1)` |

### Dates & Times

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `created_at` | datetime | When created | `created_at >= -7d` |
| `updated_at` | datetime | Last modified | `updated_at >= startOfMonth()` |
| `last_run` | datetime | Last execution | `last_run >= 2024-01-01` |
| `approved_at` | datetime | When approved | `approved_at IS NOT NULL` |

### Organization

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `folder_id` | uuid | Parent folder | `folder_id IS NOT NULL` |
| `folder_path` | string | Folder path | `folder_path ~ "/auth/"` |
| `tags` | array | Test tags | `tags CONTAINS "smoke"` |
| `component` | string | Component name | `component = "checkout"` |

### Quality Metrics

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `pass_rate` | number | 0.0 to 1.0 | `pass_rate > 0.9` |
| `execution_count` | number | Times executed | `execution_count >= 10` |
| `avg_duration` | number | Avg time (seconds) | `avg_duration < 300` |
| `failure_count` | number | Times failed | `failure_count > 0` |
| `is_flaky` | boolean | Inconsistent results | `is_flaky = true` |

### Ownership

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `created_by` | string | Creator user ID | `created_by = "user_123"` |
| `updated_by` | string | Last editor | `updated_by = "current_user"` |
| `assigned_to` | string | Assignee | `assigned_to = "current_user"` |
| `reviewer` | string | Reviewer | `reviewer IS NOT NULL` |
| `team` | string | Team name | `team = "qa"` |

### Automation

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `automated` | boolean | Is automated | `automated = true` |
| `automation_status` | string | Status | `automation_status = "in_progress"` |
| `automation_test_id` | string | External test ID | `automation_test_id ~ "selenium"` |
| `automation_framework` | string | Framework used | `automation_framework = "playwright"` |

### Requirements

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `requirements` | array | Requirement IDs | `requirements CONTAINS "US-123"` |
| `dependencies` | array | Dependent tests | `dependencies IS NOT NULL` |
| `defect_links` | array | Linked bugs | `defect_links CONTAINS "JIRA-456"` |
| `known_issues` | array | Known issues | `known_issues IS NULL` |

## Field Type Details

### String Fields

Case-insensitive by default. Use pattern operators:

```oql
title ~ "login"           # Contains
title ^ "TC-"            # Starts with
title $ "test"           # Ends with
```

### Number Fields

Support comparison and range operations:

```oql
priority < p2            # Less than
version >= 2             # Greater or equal
age_days BETWEEN 7 AND 30  # Range
```

### Boolean Fields

```oql
automated = true
is_flaky = false
```

### Datetime Fields

Support ISO dates and relative dates:

```oql
created_at = "2024-01-15"
created_at >= -7d
created_at >= startOfMonth()
```

### Array Fields

Support contains operations:

```oql
tags CONTAINS "smoke"
tags CONTAINS ANY (smoke, regression)
tags CONTAINS ALL (smoke, critical)
requirements IS NOT NULL
```
